范例程序需要用 Visual C++ 与 DirectX SDK 来编译，如果读者使用的是专业版的 Visual C++ ，应该就有内附 DirectX SDK。虽然不是完整的版本，但是已经足够用来编译范例程序。

  • http://www.microsoft.com/express/ 可以下载 Visual Studio Express Editions。

  • http://msdn.microsoft.com/en-us/directx/ 可以下载 DirectX SDK。

  • 最好是先安装 Visual C++ ，再安装 DirectX SDK 。反过来安装并不是不行，只是要自行手动设置 include files 跟 library files 的路径。




编译范例程序时，有几点需要注意。

  • Startup Project 不要设成 glib ，它是被所有范例程序共用的程序库，并不会输出运行文件。要把 Startup Project 设置成主程序所在的 Project 。

  • 每个 project/solution 都有 4 种编译模式， Debug 、 Debug_DX10 、 Release 、 Release_DX10 。字尾有 DX10 的模式才会编译 Direct3D10 的代码，非 DX10 的模式只支持 Direct3D9 与 OpenGL。想放断点来单步运行程序时，请使用 Debug 、 Debug_DX10 这两种模式。

  • 编译过程所生成的文件，会输出到 3DGraphicsProgramming/Bin 目录下。Debug、Debug_DX10 模式所生成的运行文件，也会输出到 3DGraphicsProgramming/Bin 目录里。Release、Release_DX10 模式所生成的运行文件，则是与 C++ 源代码放在相同目录下。

  • 有些 OpenGL 范例需要用到扩展功能，驱动程序若没有支持，程序有可能会崩溃。使用这些扩展功能前，理论上都要先检查硬件的支持程度，本书范例程序并不是每一个都有去仔细检查。

  • 某些 Direct3D9 范例在旧的硬件上会无法运行，或者是看到错误的结果。例如第 7 章里某些使用贴图投影计算的程序，以及第 8 章里使用 two-sided stencil buffer 功能的程序。

  • 并不是每个范例程序都正确的支持缩放窗口功能，但想要修正并不难，读者可以自行尝试。

  • 可以把显卡 OpenGL 驱动程序里的等待垂直同步功能打开，不然有些 OpenGL 范例会运行太快，鼠标和键盘操作感觉上也会变慢。
